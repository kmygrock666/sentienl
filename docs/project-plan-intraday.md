# 專案計畫書：台股「明日之星」13:00 即時選股引擎（精準策略版）

## 1. 專案目標

於交易日 13:00 啟動，抓取即時價量與盤中成交明細，識別「大戶鎖漲停」與「尾盤量能爆發」形態，記錄模擬交易並推播至 Telegram。

## 2. 核心策略邏輯（早稻田策略）

- **13:00 選股條件**：當日漲幅需達 7.5% 以上。
- **大戶對大戶（能量識別）**：
  - 觀察漲停板（Limit Up）之委賣量被吃掉的過程。
  - 超級大戶：單筆成交量（Single Trade Volume）直接單吃漲停板總委賣量。
  - 大戶：單筆成交量佔漲停板總委賣量 >= 1/3。
- **量能爆發比**：觀察 12:00 前後的量能分佈。
  - 條件：12:00 以前累計量 = (13:00 累計量 - 12:00 累計量) × 1/3（即下午盤一小時的量是早盤三小時的三倍）。
- **量縮鎖漲停**：漲停鎖住且今日成交量小於昨日成交量。
- **123 法則（建位參考）**：8.5% 買 1 張 / 9% 買 2 張 / 10% 買 3 張。
- **出場邏輯**：隔日預掛上下 2% 或 09:30 前沖銷，或於高點出場。

## 3. 整合開發階段

### 第一階段：歷史基底與勝率指標（Data Integration）

- **任務**：利用 SQLite 資料庫計算標的歷史「漲停後隔日表現」。
- **動作**：計算過去半年，個股出現「強勢收盤/漲停」後，隔日早盤的獲利機率與平均漲幅。

### 第二階段：盤中即時監控模組（Intraday Monitor）

- **任務**：串接證交所 MIS 系統。
- **動作**：
  1. 實作 `get_realtime_snapshot`。
  2. 需抓取 `tv`（單筆成交量）與 `u`（漲停價）及最佳五檔。
  3. 實作「12:00 快照」機制，於 12:00 記錄一次累計量。

### 第三階段：策略引擎（Tomorrow's Star Engine）

- **任務**：實現「大戶對大戶」與「量能比」邏輯。
- **邏輯**：判斷 13:00 後的即時單筆成交是否滿足「單吃 1/3 漲停量」。
- **隔離**：程式碼完全存放於 `sentinel/intraday/`，不干擾盤後 `pipeline.py`。

### 第四階段：自動化執行與通知（Automation & Alert）

- **任務**：整合 Telegram 回報。
- **參數**：透過環境變數 `TS_TG_TOKEN` 和 `TS_TG_CHAT_ID` 設定（請勿硬編碼至程式碼或文件）。

### 第五階段：模擬交易紀錄（Trade Tracking）

- **任務**：新增 `IntradayTrades` 表，記錄 13:00 模擬進場點，並於隔日自動計算損益。

## 4. 開發備註

- **效能**：針對昨日成交量前 300 檔進行監控。
- **穩定性**：處理 MIS API Session 保持與 Retry 機制。
