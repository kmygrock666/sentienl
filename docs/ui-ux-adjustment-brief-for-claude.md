# Sentinel UI 人類行為檢查與調整需求（給 Claude Code 實作）

> 日期：2026-05-10  
> 目的：針對已完成 UI 的可理解性與操作流做修正，避免違反使用者心智模型。

---

## 1) 結論摘要（先看這段）

目前 UI 已具備完整功能入口，但在「任務命名語意」與「執行後導流」上，與人類實際操作行為不一致：

1. `Run（今日）` 命名會誤導使用者，以為掃描「最新可用收盤交易日」，但實作其實是用「今天日期區間」送 `run`。
2. 首頁快捷任務執行成功後，使用者仍需手動切頁查結果，流程中斷。
3. 「策略命中摘要」缺乏語境，不易理解是什麼資料、代表哪次掃描、可採取什麼下一步。

以上三點是優先修正項，屬於 UX P1。

---

## 2) 現況優點（保留）

1. 功能覆蓋完整：CLI 主要能力都有 UI 入口。
2. 有 Task Center：長任務可追蹤，方向正確。
3. 有 command preview：可追溯性佳。
4. 查詢與執行分離：頁面責任清楚。

---

## 3) 現況缺點（需修正）

## 3.1 命名與實際邏輯不一致（P1）
- 位置：`ui/app.py` 快捷按鈕 `▶ Run（今日）`
- 目前行為：
  - `start-date = today - 1`
  - `end-date = today`
- 問題：
  - 「今日」是執行日，不是交易日。
  - 非交易日/收盤前可能無有效資料，使用者仍期待「最新收盤交易日」結果。

## 3.2 執行後沒有結果導流（P1）
- 位置：`ui/app.py` 快捷操作區
- 目前行為：送出任務後只顯示成功訊息，未導向結果。
- 問題：
  - 使用者目標是「看結果」，不是「送出任務」。
  - 需要跨頁自行找，造成認知負擔。

## 3.3 策略命中摘要語意不完整（P1）
- 位置：首頁「策略命中摘要（最新交易日）」
- 目前行為：僅顯示 `strategy_id -> hits` 圖表與小表。
- 問題：
  - 不知道「最新交易日」實際是哪天。
  - 不知道分母（母體股票數）與涵義（命中數非績效）。
  - 沒有 CTA（例如「查看該日完整結果」）。

## 3.4 任務成功與資料更新的關聯弱（P2）
- 任務成功後首頁未明確提示「資料已更新到哪個交易日」。
- 使用者不確定是否需要再跑一次或去哪裡確認。

---

## 4) 調整方案（請 Claude 直接實作）

## 4.1 重命名與邏輯修正：`Run（今日）` -> `Run（最新收盤日）`（P1）

### 需求
1. 首頁快捷按鈕改名：`▶ Run（最新收盤日）`。
2. 執行邏輯改為：
   - 先查詢各市場最新可用交易日（建議用 `daily_prices` / `trading_calendar` 交集，至少確保有資料）。
   - 將 `run` 的 `start-date`、`end-date`、`trading-date` 都設為該「最新收盤交易日」。
3. 若 TWSE/TPEX 最新日不同：
   - 預設採「兩市場共同可用的最小最新日」並提示原因。
   - UI 顯示：`本次以 YYYY-MM-DD 為掃描交易日（市場可用日對齊）`。

### 驗收
- 非交易日執行快捷鍵，仍能穩定掃描到最近收盤交易日。
- 按鈕名稱與實際行為一致。

## 4.2 快捷任務完成後自動導流到結果（P1）

### 需求
1. 首頁快捷任務送出後，於 session_state 記錄 `task_id` 與 `target_page`。
2. 任務完成後自動導向對應頁：
   - `run` -> `Daily Scan` 並預選 `trading_date = 本次掃描日`
   - `sync` -> `Inspect`（status）或 `Data Sync` 最新任務詳情
   - `run-intraday` -> `Intraday`（run tab）
3. 提供可關閉選項（例如 `自動開啟結果頁` toggle，預設開）。

### 驗收
- 使用者不需手動切頁即可看到結果。
- 導向後可直接看到與該 task 對應的 log/結果。

## 4.3 重做「策略命中摘要」資訊語境（P1）

### 需求
首頁摘要區塊需增加：
1. 標題改為：`策略命中摘要（最新可用掃描日）`。
2. 顯示完整上下文：
   - `掃描交易日：YYYY-MM-DD`
   - `總命中數：N`
   - `市場涵蓋：TWSE x 檔 / TPEX y 檔`（若可得）
3. 圖表註解：
   - 明確說明「此為各策略命中檔數，不代表績效或勝率」。
4. 行動按鈕（CTA）：
   - `查看該日完整名單` -> 跳 Daily Scan 並帶入該日篩選。
5. 若無資料：
   - 顯示「尚無掃描結果」+ `前往執行最新收盤日掃描` 按鈕。

### 驗收
- 新使用者可在 10 秒內理解該區塊的資料時間與含義。
- 可一鍵進入完整結果頁，不需重新選條件。

## 4.4 任務完成回饋強化（P2）

### 需求
1. 任務完成訊息加上「結果定位資訊」：
   - 例：`Run 完成：掃描交易日 2026-05-08，命中 37 檔。`
2. 在 Task Center 每筆 run 任務增加 `Open Result` 快捷。
